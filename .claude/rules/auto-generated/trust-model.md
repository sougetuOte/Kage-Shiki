# 信頼度モデル — TDD 内省パイプライン

## 観測回数による段階

| 回数 | 段階 | アクション |
|------|------|-----------|
| 1 | 記録 | `.claude/tdd-patterns.log` に記録のみ |
| 2 | 注意 | パターン詳細を `docs/memos/tdd-patterns/` に書き出し |
| 3 | 候補生成 | `draft-NNN.md` を自動生成、`/daily` で通知 |
| 3+ | 承認待ち | `/pattern-review` で PM級承認を求める |

## パターン照合ロジック

### MVP（Phase 3 導入時）

手動照合。`/pattern-review` 実行時に tdd-patterns.log を目視で確認し、
類似パターンを人間が判断する。

### 完全実装（将来）

- ファイルパスの共通プレフィックスで照合
- エラーメッセージのキーワード照合
- テスト名のパターンマッチング

## ルール候補のフォーマット

```markdown
# draft-NNN: [ルール名]

- 生成日: YYYY-MM-DD
- 観測回数: N
- ステータス: draft / approved / retired
- last_matched: YYYY-MM-DD

## 根拠パターン

| # | 日時 | テストファイル | 失敗メッセージ |
|---|------|--------------|-------------|
| 1 | ... | ... | ... |

## 推奨ルール文

[ルールの具体的な記述]

## 適用範囲

[どのファイル・モジュールに適用するか]
```

## ルール寿命管理

- `last_matched` が **90日以上前** → `/daily` で棚卸し通知
- 棚卸し候補は PM級承認で以下のいずれか:
  - **更新**: 適用範囲を見直して継続
  - **削除**: 不要と判断して retired に変更
  - **統合**: 他ルールとマージ
