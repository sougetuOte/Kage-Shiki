## Warning Issues (18 件)

### 並行性・スレッド安全性

#### [W-1] Warning [SE]: `stop()` に競合ウィンドウがある
**File**: `src/kage_shiki/agent/desire_worker.py` (stop メソッド付近)
**Source**: Agent 4 (セキュリティ)
**内容**: `stop()` は Lock を取得せずに `_running = False` + `self._timer.cancel()` を実行。`_schedule_next()` の `self._timer` への書き込みと `stop()` の `cancel()` が競合すると、古い Timer に cancel がかかり新 Timer が残存する可能性。
**Fix**: `_timer` および `_running` の読み書きを `_lock` で保護する。または `threading.Event` で停止シグナル化。

#### [W-2] Warning [SE]: `notify_user_input()` の Lock 境界が非対称
**File**: `src/kage_shiki/agent/desire_worker.py` (notify_user_input / update_desires)
**Source**: Agent 1 W-1 + Agent 4 W-1 (重複統合)
**内容**: `notify_user_input()` は Lock 非取得で `_last_user_input_time` を書き換え、`update_desires()` は Lock 外で同値を読み取る。GIL 保護で実害は限定的だが、Lock 境界が `reset_all()` (Lock あり) と `notify_user_input()` (Lock なし) で非対称。保守時の誤認リスク。
**Fix**: クラス docstring の「スレッド安全性」セクションに `notify_user_input` は GIL 保護のみ（意図的に Lock 不使用）と明示。または Lock を取得して統一。

#### [W-3] Warning [SE]: コールバック例外で Timer ループが無音停止する
**File**: `src/kage_shiki/agent/desire_worker.py` (update_desires の on_threshold_exceeded 呼び出し)
**Source**: Agent 4 I-3 (昇格)
**内容**: `self._on_threshold_exceeded(desire_type)` で例外が発生すると `update_desires()` が伝播停止し `_schedule_next()` が呼ばれず、DesireWorker が無音で死亡する。
**Fix**: コールバック呼び出しを `try/except Exception` で囲み、`logger.error(..., exc_info=True)` で記録して処理を継続する（Noisy Failure パターン）。
