## 主要な設計改善（iter を通じた成果）

### 1. `DesireWorker` の DB 直接依存を排除（R-11 カプセル化改善）
- 当初設計: `db_conn: sqlite3.Connection` を直接受け取る
- 最終設計: `get_pending_curiosity_count` / `get_observation_count` の Callable 注入
- テスタビリティが向上し、DB モック不要で単体テスト可能に

### 2. Timer 二重起動の論理的排除（並行性の決定論化）
- `_start_generation` 世代カウンタを導入
- `start()` / `stop()` で世代を進め、`update_desires()` の末尾で世代一致を確認してから schedule
- `stop → start → old callback completes` のシナリオで Timer 二重起動を排除

### 3. コールバック例外の Noisy Failure 化
- iter 0 では例外が握りつぶされ Timer ループが無音停止する可能性
- `try/except Exception` + `logger.error(exc_info=True)` で記録しつつ処理継続
- Silent Failure を Noisy Failure に昇格

### 4. 列挙値バリデーションの追加（R-2 強化）
- `AgenticSearchConfig.engine` / `search_api` に `_coerce_enum` によるホワイトリストチェック
- `_VALID_AGENTIC_ENGINES` / `_VALID_AGENTIC_SEARCH_APIS` frozenset

### 5. `update_target_status` の COALESCE SQL
- `result_summary=None` による既存値 NULL 上書きバグを修正
- `SET result_summary=COALESCE(?, result_summary)` でセマンティクス明確化

### 6. Wave 2 ブロッカー解消
- Phase 2b Section 7.3 の 4 purpose (`autonomous_talk`, `agentic_decompose`, `agentic_summarize`, `agentic_noise`) が未接続だった問題を iter 0 で解消
- Wave 2 の `handle_autonomous_turn()` 実装時に `ValueError: Unknown purpose` で即失敗する事態を回避
