# Session Management

本ドキュメントは、セッション状態の保存・復元とコンテキスト管理のプロトコルを定義する。

## 1. SESSION_STATE.md

セッション間のハンドオフに使用するファイル。プロジェクトルートに配置する。

### 構造

```markdown
# SESSION_STATE

**セッション日時**: YYYY-MM-DD
**フェーズ**: PLANNING / BUILDING / AUDITING

---

## 完了タスク
- [完了した作業の要約]

## 進行中タスク
- [作業中の内容と進捗]

## 次のステップ
1. [次に行うべき作業（優先順位付き）]

## 未解決の問題
- [ブロッカーや据え置き事項]

## コンテキスト情報
- フェーズ: [現在のフェーズ]
- ブランチ: [ブランチ名と状態]
- Python: [バージョン]
- テスト: [passed 数 / カバレッジ]
- ruff: [状態]
- [参照すべきドキュメントのパス一覧]
```

### 運用ルール

- SESSION_STATE.md は「使い捨て」であり、毎回上書きする
- git 管理対象外（`.gitignore` に含む）
- SSOT（`docs/internal/`, `docs/specs/`）の代替にはならない

## 2. Save / Load コマンド

### `/quick-save`（軽量版）

- **用途**: 日常の区切り、コンテキスト残量が少ないとき
- **動作**: SESSION_STATE.md のみ記録
- **コンテキスト消費**: 3-4%

### `/quick-load`（軽量版）

- **用途**: 日常の再開
- **動作**: SESSION_STATE.md を読み込み、復帰サマリーを報告
- **コンテキスト消費**: ~1%

### `/full-save`（完全版）

- **用途**: 一日の終わり、長期中断前
- **動作**: SESSION_STATE.md 記録 + git commit + push + `/daily` 振り返り
- **コンテキスト消費**: ~10%
- **前提**: コンテキスト残量に余裕があること（25% 以上推奨）

### `/full-load`（完全版）

- **用途**: 数日ぶりの復帰、大規模な状態確認
- **動作**: SESSION_STATE.md 読み込み + 関連ドキュメント特定 + 詳細な復帰報告
- **コンテキスト消費**: 2-3%
- **注意**: ドキュメントの先読みは行わない（作業開始時に読み込む）

### 使い分けフロー

```
日常の中断 → /quick-save → /quick-load で再開
一日の終わり → /full-save（余裕あれば）or /quick-save
数日ぶり復帰 → /full-load
```

## 3. コンテキスト残量管理

### 残量目安表

| 残量 | 状態 | 推奨アクション |
|:-----|:-----|:--------------|
| > 30% | 安全 | 通常作業を継続 |
| 15-30% | 注意 | 区切りのよいところで `/quick-save` を提案 |
| <= 15% | 危険 | 即座に `/quick-save`。`/full-save` は禁止 |

### ルール

- コンテキスト残量 **20% を下回った** と判断したら、ユーザーに `/quick-save` を提案する
- auto-compact の発動を待たない（保険として先回りする）
- 残量 25% 以下では `/full-save` を使わない（消費が大きいため）
- 残量 15% 未満で `/full-save` が要求された場合、`/quick-save` にフォールバックする

## 4. Context Compression（コンテキスト圧縮）

セッションが長期化し、コンテキストが肥大化した場合の手順:

1. **決定事項の書き出し**: 未記録の決定事項を `docs/tasks/` または `docs/memos/` に書き出す
2. **状態の保存**: `/quick-save` でセッション状態を保存
3. **リセット宣言**: ユーザーに「コンテキストをリセットします」と宣言
4. **新セッション開始**: `/quick-load` で最小限のコンテキストで再開

### 圧縮の判断基準

- 同一セッションで 3 Wave 以上を処理した場合
- auto-compact が複数回発動した場合
- ツール呼び出しの応答が遅くなった場合
