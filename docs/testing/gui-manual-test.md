# GUI 手動テストチェックリスト

**作成日**: 2026-03-06
**対応 FR**: FR-8.2
**関連タスク**: T-27
**対象バージョン**: Phase 2a 以降

---

## 概要

このドキュメントは、影式 (Kage-Shiki) の GUI 変更時に実施する手動テストのチェックリストを定義する。
教訓 L-5「GUI のレイアウト崩れを目視確認していなかった」（`docs/memos/hotfix-runtime-bugs-phase1.md`）を踏まえ、
自動テストで検証できない視覚的・操作的な品質項目を網羅する。

**実施タイミング**:
- GUI レイアウト・ウィジェットを変更したとき（`.claude/rules/building-checklist.md` R-10 参照）
- Phase 完了前の最終確認時（スモークテスト手順書と合わせて実施）

---

## 前提条件

- スモークテスト手順書（`docs/testing/smoke-test.md`）のセクション 1 が完了し、アプリケーションが起動していること
- 初回起動の場合はウィザードが完了していること（`persona_frozen = true` の状態）

---

## チェック項目 1: 長文応答のスクロール表示

**目的**: 200 文字超の応答がスクロール可能な Text ウィジェットに正しく表示されることを確認する

- [ ] 長文応答を誘発する入力を送信する
  - **手順**: 「詳しく教えてください」「長い話をしてください」等、200文字以上の応答が得られる入力をする
  - **期待結果**: 応答テキストが Text ウィジェットに表示される

- [ ] スクロールが可能であることを確認する
  - **手順**: テキスト表示エリアにマウスカーソルを置き、マウスホイールを回す
  - **期待結果**: テキストがスクロールする。最初と最後のテキストが表示エリアの外にはみ出さず、スクロールで閲覧できる

- [ ] テキストがウィンドウ領域の外にはみ出さないことを確認する
  - **手順**: 長文応答表示後のウィンドウ全体を目視確認する
  - **期待結果**: テキストがウィンドウ枠の外にはみ出していない。Scrollbar ウィジェットが適切に機能している

- [ ] 複数回の応答後もスクロール表示が正常であることを確認する
  - **手順**: 3回以上メッセージを送受信する
  - **期待結果**: テキスト表示エリアのレイアウトが崩れない。最新の応答が表示されている

---

## チェック項目 2: ウィンドウドラッグ移動

**目的**: タイトルバーのない枠なしウィンドウ（`overrideredirect(True)`）をドラッグ操作で任意位置に移動できることを確認する

- [ ] ウィンドウ本体をドラッグして移動できることを確認する
  - **手順**: ウィンドウのテキスト表示エリアまたはキャラクター表示領域（タイトルバーではない）をマウス左ボタンで押したまま、デスクトップ上の別の位置までドラッグして離す
  - **期待結果**: ウィンドウがドラッグした位置に移動する

- [ ] 画面端近くへの移動が正常であることを確認する
  - **手順**: ウィンドウをデスクトップの端（左上・右上・左下・右下）付近までドラッグする
  - **期待結果**: ウィンドウが画面外に消えない。ドラッグ後もウィンドウが操作可能な状態にある

- [ ] 移動後にウィンドウが正常に動作することを確認する
  - **手順**: ウィンドウを移動後、入力欄にテキストを入力して送信する
  - **期待結果**: 移動後も入力・送信・応答表示が正常に動作する

---

## チェック項目 3: トレイ最小化・復帰

**目的**: トレイアイコン操作でウィンドウの表示・非表示が切り替わることを確認する

- [ ] トレイアイコンの左クリックでウィンドウが非表示になることを確認する
  - **手順**: タスクバーのトレイエリアにある影式のアイコンを左クリックする
  - **期待結果**: デスクトップ上のウィンドウが非表示（`withdraw()`）になる。タスクバーにも表示されない

- [ ] トレイアイコンの再クリックでウィンドウが再表示されることを確認する
  - **手順**: ウィンドウが非表示の状態で、トレイアイコンを再度左クリックする
  - **期待結果**: 非表示になっていたウィンドウが再表示（`deiconify()`）される。ウィンドウは直前と同じ位置に表示される

- [ ] 最小化・復帰を複数回繰り返しても正常であることを確認する
  - **手順**: トレイアイコンの左クリックによる非表示・再表示を 3 回繰り返す
  - **期待結果**: 毎回正常に表示・非表示が切り替わる。ウィンドウのレイアウトが崩れない

- [ ] 非表示中もアプリケーションが動作していることを確認する
  - **手順**: ウィンドウを非表示にしてから 10 秒待ち、その後再表示する
  - **期待結果**: 再表示後もアプリケーションが正常に動作する。トレイアイコンは表示されたままである

---

## チェック項目 4: ウィザード画面遷移

**目的**: 人格生成ウィザードの全フロー（方式選択 → 入力 → プレビュー → 確定）が目視で正常に動作することを確認する

> **前提**: このチェック項目は `config.toml` の `persona_frozen = false` の状態で実施する。
> 確認後は `persona_frozen = true` に戻すか、テスト用 `data_dir` を使用すること。

### 4-1. 方式選択画面

- [ ] ウィザード起動時に方式選択画面が表示されることを確認する
  - **手順**: `persona_frozen = false` の状態で `python -m kage_shiki` を起動する（または「ウィザード起動」をトレイメニューから選択する）
  - **期待結果**: 3 方式（「AI おまかせ」「既存イメージ」「白紙育成」）のボタンを持つ画面が表示される

- [ ] 各方式ボタンが機能することを確認する
  - **手順**: いずれかの方式（例: 「AI おまかせ」）のボタンをクリックする
  - **期待結果**: 選択した方式に対応する入力画面に遷移する

### 4-2. 入力画面

- [ ] 入力画面が方式に応じて正しく表示されることを確認する
  - **手順（方式 A: AI おまかせ）**: キーワード入力欄が表示されることを確認する
  - **手順（方式 B: 既存イメージ）**: 自由記述入力欄が表示されることを確認する
  - **手順（方式 C: 白紙育成）**: 名前・一人称の入力欄が表示されることを確認する
  - **期待結果**: 各方式に対応した入力フォームが表示されている

- [ ] 入力欄にテキストを入力して次へ進めることを確認する
  - **手順**: 入力欄に適切なテキストを入力し、「次へ」または送信ボタンをクリックする
  - **期待結果**: 生成中の表示（ドットアニメーション等）を経て、プレビュー会話画面に遷移する

### 4-3. プレビュー会話画面

- [ ] プレビュー会話画面が正しく表示されることを確認する
  - **手順**: 入力画面から遷移後の画面を目視確認する
  - **期待結果**: キャラクター名、生成されたセリフ、入力欄、「送信」ボタン、「確定する」ボタン、「やり直す」ボタンが表示されている

- [ ] プレビュー会話 UI で入力・送信が機能することを確認する
  - **手順**: プレビュー会話の入力欄にテキストを入力し、「送信」ボタンをクリックする
  - **期待結果**: 送信後、生成された人格のキャラクターがセリフを返す。表示エリアに返答が追加される

- [ ] 「やり直す」ボタンで方式選択画面に戻ることを確認する
  - **手順**: 「やり直す」ボタンをクリックする
  - **期待結果**: 方式選択画面（4-1）に戻る

### 4-4. 凍結確認ダイアログと確定

- [ ] 「確定する」ボタンで凍結確認ダイアログが表示されることを確認する
  - **手順**: プレビュー会話画面の「確定する」ボタンをクリックする
  - **期待結果**: 「このキャラクターで始めますか？」等の確認ダイアログが表示される。「はい、始める」と「キャンセル」の選択肢がある

- [ ] 「キャンセル」でプレビュー会話画面に戻ることを確認する
  - **手順**: 凍結確認ダイアログの「キャンセル」をクリックする
  - **期待結果**: プレビュー会話画面に戻る

- [ ] 「はい、始める」で人格が凍結されウィザードが完了することを確認する
  - **手順**: 凍結確認ダイアログの「はい、始める」をクリックする
  - **期待結果**:
    - `~/.kage_shiki/persona_core.md` が生成されている
    - `~/.kage_shiki/style_samples.md` が生成されている
    - `~/.kage_shiki/config.toml` の `persona_frozen` が `true` に更新されている
    - ウィザードが完了してプロセスが終了する（次回起動で通常対話モードに入る）

## 追加確認項目（任意）

以下の項目は必須ではないが、GUI 変更が多い場合に追加で確認することを推奨する。

- [ ] **ウィンドウリサイズ時のレイアウト**: ウィンドウサイズを変更した場合（`overrideredirect` が解除された状態でのみ適用可能）にレイアウトが崩れないこと
- [ ] **日本語入力の確認**: IME を使用した日本語入力が入力欄で正常に機能すること
- [ ] **長いキャラクター名の表示**: キャラクター名が長い場合（10文字以上）でも UI が崩れないこと

---

## 合否判定

全チェック項目が通過した場合のみ、GUI 変更が完了したとみなす。
失敗項目がある場合は `.claude/rules/building-checklist.md` の R-10 に従い修正すること。

| チェック項目 | 確認項目数 | 合否 |
|-----------|----------|------|
| 1. 長文応答のスクロール表示 | 4 | - |
| 2. ウィンドウドラッグ移動 | 3 | - |
| 3. トレイ最小化・復帰 | 4 | - |
| 4. ウィザード画面遷移 | 9 | - |
| **合計（必須）** | **20** | - |
