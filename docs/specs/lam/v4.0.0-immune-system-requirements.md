# 免疫系アーキテクチャ要件定義（LAM v4.0.0）

## 概要

LAM v4.0.0 の中核コンセプト「免疫系アーキテクチャ」の要件定義。
プロジェクトの品質を自動的に維持・回復する仕組みを構築する。

## 機能要求

### FR-IS-001: 権限等級システム
全ての変更を PG/SE/PM の 3 段階にリスク分類し、等級に応じた修正制御を行う。

### FR-IS-002: 自動ループ統合
`/full-review` の結果に基づき、Green State 未達の場合に自動的にイテレーションを繰り返す。

### FR-IS-003: TDD 内省パイプライン
テスト失敗→成功のパターンを自動記録し、閾値到達時にルール候補を自動生成する。

### FR-IS-004: ドキュメント自動追従
コード変更時に影響を受けるドキュメントを自動検出し、更新を促す。
- **受け入れ条件**: `src/` 配下の `.py` ファイルが変更された場合、PostToolUse hook が `.claude/doc-sync-flag` を生成すること。doc-writer がフラグを読み取り、対応ドキュメントを特定して更新提案を出力すること。

### FR-IS-005: Green State 収束条件
5 つの条件（テスト/lint/Issue/仕様差分/セキュリティ）で品質を定量判定する。

## 非機能要求

### NFR-IS-001: 段階的導入
hooks 未導入環境でも手動フォールバックで動作すること。

### NFR-IS-002: コンテキスト効率
自動化による追加のコンテキスト消費は最小限に抑えること。
